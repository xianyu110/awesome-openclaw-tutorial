## &#x20;**为什么要建这个群&#x20;**

我做 AI Go Code（aigocode.com）快半年了，一直是工具型产品，帮中国开发者直连 Claude、Codex、GPT 这些海外 AI 模型，按量计费。

产品跑得动，但有个问题：用户用完就走，没有粘性。

我最近在玩 OpenClaw（一个开源 AI 助理框架），发现一个有意思的事：很多开发者不是不想用这些工具，而是配置门槛太高，卡在第一步就放弃了。

如果我能把这些人聚到一起，互相帮忙踩坑、分享玩法、顺便用上我的 API 服务，这不就是一个自然的飞轮吗？

## &#x20;**定价：为什么是 ¥49&#x20;**

说实话，我纠结了大概 10 分钟。

一开始想搞阶梯定价：前 50 人免费、50-100 人 ¥29、100 人后 ¥49。听起来很科学对吧？后 来想了想， 太复杂了，直接 ¥49。

理由很简单。¥49 是过滤器，不是门票，我不想做客服群，不想每天被"这个怎么配置"的问题淹没，愿意掏 49 块的人，至少是认真想玩的同时送 $50 API 额度，用户付 ¥49（约 $7），拿到 $50 额度，数学上他赚了，心理上没有"花钱买了个群"的感觉。而这 50 额度会导流到 aigocode.com，用户注册、试用、习惯了之后自然续费。红包是获客成本，额度是钩子，API 消费是 LTV。

## &#x20;**基建：1 天搭了什么&#x20;**

光有个群不够，得让用户进来就知道干什么、怎么开始。我花了一天搭了这些东西。

写了一份上手教程，Notion 文档，5 步从零跑通 OpenClaw + AI Go Code：获取 API Key → 安装 → 接入模型 → 连 Telegram → 切换模型。新手照着做，5 分钟搞定。

群公告写了 2 个版本，不同平台用不同版本。直接讲清楚：这不是客服群，不是课程群，没有老师没有助教没有答疑义务。入群 = 认同以上。这句话很重要，降低预期就是降低运营成本。我一个人做不了客服，也不想做。

自动化付款+进群是关键一环。刚开始发出去没人理，后来小伙伴在高铁上用 Claude Max 调了一版新海报，换了海报加上微信支付自动化，扫码付款后自动弹出群二维码，立刻就有人开始进了。不得不说团队太给力了。这里的关键是降低用户操作成本，从"看到→付款→进群"这条链路越短、越顺滑，转化率越高。
